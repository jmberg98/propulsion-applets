SECTION 1 — PROJECT IDENTITY
________________________________________________________________________________

This codebase produces self-contained educational applets for MIT-level space
propulsion and plasma physics courses.

Each module consists of the following files (all in the SAME directory):

  FILE                   ROLE
  --------------------   ------------------------------------------------------
  larmor.py              Python prototype (local HTTP server + embedded HTML)
  larmor_circle.html     Standalone applet — MODE = circle
  larmor_single.html     Standalone applet — MODE = one
  larmor_double.html     Standalone applet — MODE = double
  larmor_applet.html     Webpage that embeds all three applets via <iframe>

All files for a module live in the same directory. Applets are embedded as
same-origin iframes; cross-origin restrictions do not apply.

SECTION 3 — DEVELOPER WORKFLOW — STAGE BY STAGE
________________________________________________________________________________

STAGE 1 — PYTHON PROTOTYPE
---------------------------
[DEV]
Build and iterate the applet logic as <module>_applet.py. This gives instant
feedback without rebuilding HTML.

[AGENT]
When asked to generate a Python prototype:
  - Embed the full HTML/JS as a multiline string:   HTML = r"""..."""
  - Use ThreadingMixIn + HTTPServer
  - Expose a single /api/compute POST endpoint that accepts JSON and
    returns JSON
  - The compute function raises ValueError for bad inputs; the handler
    returns {"error": "..."} on exception
  - The browser-side JS does:
      fetch('/api/compute', { method:'POST', ... })
    and awaits the result
  - Keep the prototype's panel simpler than the final HTML
    (no sliders needed, just <input type="number">)


STAGE 2 — STANDALONE HTML APPLET
----------------------------------
[DEV]
Once the physics and visualization are correct in the prototype, convert to a
standalone HTML file. The compute logic moves fully into the browser.

[AGENT]
When converting prototype to standalone applet:
  - Replace all fetch('/api/compute') calls with a local compute(...) function
    that throws Error (not ValueError)
  - Add the full panel design (sliders, preset dropdowns, display toggles,
    output table, legend)
  - Add the quiz overlay system (see Section 10)
  - Add the MODE and QUIZ_COMPLETED constants at the top of the
    <script type="module"> block
  - One file per MODE variant (circle / single / double)
  - Only MODE and QUIZ_COMPLETED differ between variant files


STAGE 3 — WEBPAGE ASSEMBLY
----------------------------
[DEV]
Provide the Webpage Content Doc. Claude assembles the HTML page from
components in Section 5.

[AGENT]
When building the webpage:
  - Copy explanatory text VERBATIM from the content document — do not
    paraphrase
  - Each quiz question immediately precedes the demo it unlocks
  - Derivations go inside details.deriv-detail inside the relevant
    theory-block
  - The footer reads:  Created by [Name] — MIT Propulsion — [Year]

4.7  MATH RENDERING
--------------------
  Library:  KaTeX 0.16.9  (CDN — exact URLs in Section 2)

  Delimiters registered in auto-render config:
    Display block:   $$...$$   or   \[...\]
    Inline:          $...$     or   \(...\)

  Font sizes (relative to parent element):
    KaTeX inline:    font-size: 1.25em
    KaTeX display:   font-size: 1.20em

  Coloring equations:
    Use \textcolor{#RRGGBB}{...} with the physics-quantity hex codes.
    \textcolor{#FF4400}{...}   thrust / mass
    \textcolor{#3DBFFF}{...}   velocity / r_L
    \textcolor{#C060FF}{...}   charge q
    \textcolor{#00DDB5}{...}   B field / pressure

  Boxed results: use \boxed{...} for final derived expressions.
  Rule: the boxed result appears ABOVE the details.deriv-detail element
  so it remains visible without expanding the derivation.

SECTION 8 — PHYSICS & COMPUTE LAYER
________________________________________________________________________________

[AGENT]
This function must be IDENTICAL in the Python prototype and all HTML applets.
Never approximate or simplify the physics.

  const AMU = 1.66053906660e-27;
  const E_C = 1.602176634e-19;

  function compute(m_amu, q_e, v, theta_deg, B) {

    // Validate: all inputs must be finite numbers
    // Validate: m_amu > 0, v >= 0, B >= 0

    const m      = m_amu * AMU;
    const q      = q_e   * E_C;
    const th     = theta_deg * Math.PI / 180;
    const v_par  = v * Math.cos(th);
    const v_perp = v * Math.sin(th);
    const absq   = Math.abs(q);
    const signQ  = q > 0 ? 1 : (q < 0 ? -1 : 0);

    // Edge case: particle at rest
    if (v === 0) {
      return { v_par:0, v_perp:0, r_L:null, omega_c:null, mu:null,
               T_c:null, pitch:null, sign:signQ,
               atRest:true, pitchLine:false };
    }

    // Edge case: no charge or no field → straight line
    if (absq === 0 || B === 0) {
      return { v_par, v_perp, r_L:null, omega_c:null, mu:null,
               T_c:null, pitch:null, sign:signQ,
               atRest:false, pitchLine:true };
    }

    const omega_c = absq * B / m;
    const r_L     = v_perp ? m * v_perp / (absq * B) : 0.0;
    const mu      = m * v_perp * v_perp / (2.0 * B);
    const T_c     = 2.0 * Math.PI / omega_c;
    const pitch   = v_par * T_c;

    return { v_par, v_perp, r_L, omega_c, mu, T_c, pitch,
             sign:signQ, atRest:false, pitchLine:false };
  }

OUTPUT FIELDS:
  FIELD     UNIT    PANEL LABEL
  --------- ------- -----------------------
  r_L       m       Larmor Radius
  omega_c   rad/s   Cyclotron Frequency
  mu        J/T     Magnetic Moment
  v_par     m/s     Parallel Velocity
  v_perp    m/s     Perpendicular Velocity

NUMBER FORMATTING:
  fmt(x): returns scientific notation if |x| >= 1e4 or |x| < 1e-2,
          otherwise toPrecision(4).
  Display as "1.234\u2009m/s"  (thin-space U+2009 before unit).
  Null / undefined / NaN  →  "—"  (em dash U+2014).
